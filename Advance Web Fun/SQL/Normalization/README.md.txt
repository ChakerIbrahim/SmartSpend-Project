Overview

In this assignment, we were given an ERD that violates multiple normalization rules.
The goal is to redesign the database so that it satisfies:

First Normal Form (1NF)
Second Normal Form (2NF)
Third Normal Form (3NF)

The system is used to track students, their dojo, and their interests.

🧠 Key Learning
Never store multiple values in one column (like interests TEXT)
Use junction tables for many-to-many relationships
Always aim for clean, scalable database design
